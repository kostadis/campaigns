# DM Guide: Running Faction Assets

This guide is the practical procedure for running `Faction_Asset_Network.md` with the player-facing table in `Gauntlgrym_Asset_Table.md`.

## 1) What This System Is For

Use faction assets to replace army management with meaningful choices:
- who helps,
- what it costs now,
- what gets exposed when plans go loud.

If a choice does not create pressure, consequence, or leverage, it is not an asset decision.

## 2) What to Prep Before Session (5 minutes)

Prepare these 5 lines on your notes page:

1. Current Heat: `Ghost / Rumors / Purge / Lockdown / Extermination`
2. Active factions this session (3-5 max)
3. Which assets are `Ready`, `Strained`, `Burned`
4. One immediate ask per active faction
5. One named NPC likely to suffer fallout

Keep prep short. The system works because the costs are clear, not because prep is large.

## 3) How to Offer Assets to Players

At the negotiation scene, say:

"You can take up to two assets this operation: one for entry, one for pivot.  
Each one has a price now or a liability in-city. Choose what you are willing to carry."

Then present only relevant options (usually 4-6, not all 10).

## 4) Resolution Loop (Use Every Operation)

1. Party chooses up to 2 assets.
2. Party accepts each activation cost/liability.
3. Asset effect applies.
4. Run mission scene.
5. Check degradation by Heat.
6. Apply fallout and record one consequence in notes.

Do not skip Step 2 or Step 6. Those are where the drama lives.

## 5) Degradation Rules (Ready -> Strained -> Burned)

When an asset is used, check exposure:
- Ghost/Rumors: 1-in-6 chance to become `Strained`
- Purge/Lockdown: 3-in-6 chance to become `Strained`
- Extermination: auto-`Strained`; if used again while `Strained`, it becomes `Burned`

State meanings:
- `Ready`: full effect, normal risk
- `Strained`: usable, but add one complication
- `Burned`: unavailable until recovered/replaced by a mission outcome

## 6) How to Run Strained Assets

If a `Strained` asset is used, add one complication immediately:
- contact is tailed,
- window is shorter than promised,
- route is partially compromised,
- collateral visibility increases,
- an ally cell suffers a raid after extraction.

Complications should escalate pressure, not nullify the player's successful choice.

## 7) Cost / Liability Adjudication

Use this standard:
- Upfront price is paid now, no bargaining after effect triggers.
- Liability must appear on-screen during the arc.
- Objective lock must force mission ordering before finale.

If players avoid all costs, the faction withholds the asset. No soft freebies.

## 8) Fallout Engine (After Each Mission)

After mission resolution, apply one fallout result:
- Asset Fallout: a contact is exposed, injured, missing, or relocated
- Civic Fallout: curfew, raid, shortages, checkpoints, propaganda surge
- Credibility Fallout: faction trust drops until party addresses a demand

Pick one primary fallout and one optional minor echo.

## 9) Finale Conversion

At finale, count coalition strength:
- each `Ready` asset contributes 1 Collapse Clock segment
- each `Strained` asset contributes only if party accepts collateral consequence
- each `Burned` asset contributes 0

Outcome bands:
- 2 segments: partial success, severe survivor cost
- 4 segments: Orthodoxy command fractures
- 6 segments: controlled collapse window

## 10) Fast Rulings and Edge Cases

- Party asks for 3+ assets: allow 2 only, unless they accept instant +1 Heat and an extra liability.
- Party wants no assets: legal; increase mission difficulty and reduce extraction options.
- Party refuses payment after use: asset becomes `Burned` and faction turns hostile/passive.
- Players plan brilliantly: reward with reduced fallout severity, not removed consequences.

## 11) Table Script (30 seconds)

"Here is your operational market.  
Pick one way in and one way out.  
Every tool has a price now, not later.  
If you want clean hands, you get fewer doors."
